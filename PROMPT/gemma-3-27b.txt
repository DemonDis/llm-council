## Аналитический отчет: Сравнение фреймворков для промт-инжиниринга

**Дата:** 2026-01-19
**Автор:** Системный аналитик (опыт > 10 лет)
**Получатель:** Руководитель ИТ-подразделения

**Введение**

Настоящий отчет представляет собой анализ ключевых фреймворков для промт-инжиниринга, предназначенный для определения наиболее подходящих подходов к взаимодействию с большими языковыми моделями (LLM) в рамках ИТ-подразделения. Эффективное формирование промптов является критически важным для получения точных, релевантных и полезных ответов от LLM, что напрямую влияет на производительность и качество автоматизированных процессов. В отчете рассмотрены фреймворки RTF, COSTAR, IEEI, APE, PECRA, OSCAR и TAG, их сильные и слабые стороны, а также рекомендации по их применению.

**Основная часть**

**1. RTF (Role, Task, Format)**

*   **Описание:** Один из самых простых и распространенных фреймворков. Он фокусируется на определении роли, которую должна играть LLM, конкретной задачи, которую необходимо выполнить, и желаемого формата ответа.
*   **Правила использования:**
    *   **Role:** Укажите, кем должна быть LLM (например, "Ты - опытный программист").
    *   **Task:** Четко сформулируйте задачу (например, "Напиши функцию на Python для сортировки списка").
    *   **Format:** Определите желаемый формат ответа (например, "Предоставь код в формате Markdown").
*   **Пример промпта:** "Ты - опытный копирайтер. Напиши рекламный слоган для нового фитнес-браслета. Предоставь 3 варианта в формате списка."
*   **Плюсы:** Простота, легкость в освоении, подходит для широкого спектра задач.
*   **Минусы:** Может быть недостаточно детализированным для сложных задач, требующих более тонкой настройки.

**2. COSTAR (Context, Objective, Style, Tone, Audience, Response)**

*   **Описание:** Более комплексный фреймворк, учитывающий контекст, цель, стиль, тон, целевую аудиторию и желаемый ответ.
*   **Правила использования:**
    *   **Context:** Предоставьте необходимую информацию для понимания задачи.
    *   **Objective:** Определите цель, которую должен достичь ответ.
    *   **Style:** Укажите желаемый стиль написания (например, формальный, неформальный).
    *   **Tone:** Определите тон ответа (например, дружелюбный, профессиональный).
    *   **Audience:** Укажите целевую аудиторию.
    *   **Response:** Опишите желаемый формат и содержание ответа.
*   **Пример промпта:** "Контекст: Мы разрабатываем новый мобильный банк. Цель: Написать текст для push-уведомления о новой функции. Стиль: Неформальный, дружелюбный. Тон: Позитивный, воодушевляющий. Аудитория: Молодые люди 18-25 лет. Ответ: Напиши короткое push-уведомление, которое привлечет внимание и побудит пользователей попробовать новую функцию."
*   **Плюсы:** Высокая детализация, позволяет получить ответы, максимально соответствующие требованиям.
*   **Минусы:** Требует больше времени и усилий для формирования промпта.

**3. IEEI (Information, Example, Explanation, Instruction)**

*   **Описание:** Фреймворк, основанный на предоставлении информации, примеров, объяснений и инструкций.
*   **Правила использования:**
    *   **Information:** Предоставьте необходимую информацию для понимания задачи.
    *   **Example:** Приведите пример желаемого результата.
    *   **Explanation:** Объясните, как LLM должна выполнить задачу.
    *   **Instruction:** Дайте четкие инструкции.
*   **Пример промпта:** "Информация: Нам нужно сгенерировать описание товара для интернет-магазина. Пример: 'Умные часы X - это стильный и функциональный аксессуар, который поможет вам следить за своим здоровьем и оставаться на связи.' Объяснение: Описание должно быть кратким, информативным и привлекательным для покупателей. Инструкция: Напиши описание для беспроводных наушников Y."
*   **Плюсы:** Эффективен для обучения LLM новым задачам, особенно когда нет четких правил.
*   **Минусы:** Может быть избыточным для простых задач.

**4. APE (Action, Purpose, Expectation)**

*   **Описание:** Фреймворк, фокусирующийся на действии, цели и ожиданиях.
*   **Правила использования:**
    *   **Action:** Укажите, что LLM должна сделать.
    *   **Purpose:** Объясните, зачем это нужно сделать.
    *   **Expectation:** Опишите, что вы ожидаете получить в результате.
*   **Пример промпта:** "Действие: Напиши электронное письмо. Цель: Попросить клиента предоставить обратную связь о продукте. Ожидание: Письмо должно быть вежливым, профессиональным и содержать четкий призыв к действию."
*   **Плюсы:** Простота и ясность, подходит для задач, требующих конкретных действий.
*   **Минусы:** Может быть недостаточно детализированным для сложных задач.

**5. PECRA (Purpose, Expectation, Context, Request, Audience)**

*   **Описание:** Расширенная версия APE, включающая контекст и целевую аудиторию.
*   **Правила использования:** Аналогичны COSTAR, но с акцентом на четкое определение цели и ожиданий.
*   **Пример промпта:** "Цель: Создать пост для социальной сети, привлекающий внимание к новой акции. Ожидание: Пост должен быть креативным, информативным и содержать ссылку на акцию. Контекст: Акция действует до конца недели. Запрос: Напиши пост для Instagram. Аудитория: Молодые люди 18-25 лет."
*   **Плюсы:** Высокая детализация, позволяет получить ответы, максимально соответствующие требованиям.
*   **Минусы:** Требует больше времени и усилий для формирования промпта.

**6. OSCAR (Objective, Situation, Context, Action, Result)**

*   **Описание:** Фреймворк, используемый для решения проблем и планирования действий.
*   **Правила использования:**
    *   **Objective:** Определите цель.
    *   **Situation:** Опишите текущую ситуацию.
    *   **Context:** Предоставьте необходимую информацию.
    *   **Action:** Укажите, что нужно сделать.
    *   **Result:** Опишите желаемый результат.
*   **Пример промпта:** "Цель: Улучшить показатели конверсии на сайте. Ситуация: Показатели конверсии снизились на 10% за последний месяц. Контекст: Мы запустили новую рекламную кампанию. Действие: Проанализируй данные и предложи рекомендации по улучшению конверсии. Результат: Получить список конкретных действий, которые помогут увеличить конверсию."
*   **Плюсы:** Подходит для решения сложных проблем и планирования действий.
*   **Минусы:** Может быть избыточным для простых задач.

**7. TAG (Task, Action, Goal)**

*   **Описание:**  Фреймворк, ориентированный на определение задачи, действий и цели.
*   **Правила использования:**
    *   **Task:** Опишите задачу, которую необходимо выполнить.
    *   **Action:** Укажите конкретные действия, которые необходимо предпринять.
    *   **Goal:** Определите желаемый результат.
*   **Пример промпта:** "Задача: Создать отчет о продажах за последний квартал. Действие: Собери данные из CRM-системы и Excel-таблиц. Цель: Получить отчет, содержащий информацию о выручке, прибыли и количестве проданных товаров."
*   **Плюсы:** Простота и ясность, подходит для задач, требующих четкого определения целей и действий.
*   **Минусы:** Может быть недостаточно детализированным для сложных задач.

**Выводы**

Выбор фреймворка для промт-инжиниринга зависит от сложности задачи и требуемой степени детализации.

*   **RTF, APE и TAG** подходят для простых задач, требующих быстрого получения результатов.
*   **COSTAR и PECRA** рекомендуется использовать для сложных задач, требующих высокой детализации и соответствия конкретным требованиям.
*   **IEEI** эффективен для обучения LLM новым задачам, особенно когда нет четких правил.
*   **OSCAR** полезен для решения проблем и планирования действий.

Рекомендуется начать с простых фреймворков (RTF, APE, TAG) и переходить к более сложным (COSTAR, PECRA, OSCAR) по мере необходимости. Важно экспериментировать с различными фреймворками и адаптировать их под конкретные задачи.
